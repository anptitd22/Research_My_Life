#### File-Per-Table Tablespace Configuration

Theo mặc định, InnoDB tạo các bảng trong file-per-table tablespaces. Hành vi này được kiểm soát bởi biến `innodb_file_per_table`. Việc vô hiệu hóa `innodb_file_per_table` khiến InnoDB tạo các bảng trong system tablespace.

Bạn có thể chỉ định thiết lập `innodb_file_per_table` trong tệp tùy chọn hoặc cấu hình trong quá trình chạy bằng câu lệnh `SET GLOBAL`. Việc thay đổi thiết lập trong quá trình chạy yêu cầu quyền đủ để thiết lập các biến hệ thống toàn cục. Xem [Section 7.1.9.1, “System Variable Privileges”](https://dev.mysql.com/doc/refman/8.4/en/system-variable-privileges.html "7.1.9.1 System Variable Privileges").

Option file:

```ini
[mysqld]
innodb_file_per_table=ON
```

Using [`SET GLOBAL`](https://dev.mysql.com/doc/refman/8.4/en/set-variable.html "15.7.6.1 SET Syntax for Variable Assignment") at runtime:

```sql
mysql> SET GLOBAL innodb_file_per_table=ON;
```

#### File-Per-Table Tablespace Data Files

Một tablespace "file-per-table" được tạo trong một tệp dữ liệu .ibd trong thư mục schema nằm dưới thư mục dữ liệu MySQL. Tệp .ibd được đặt tên theo tên bảng (table_name.ibd). Ví dụ, tệp dữ liệu cho bảng test.t1 được tạo trong thư mục test nằm dưới thư mục dữ liệu MySQL.

```sql
mysql> USE test;

mysql> CREATE TABLE t1 (
    ->     id INT PRIMARY KEY AUTO_INCREMENT,
    ->     name VARCHAR(100)
    ->     ) ENGINE = InnoDB;

mysql> EXIT;
```

```terminal
$> cd /path/to/mysql/data/test
$> ls
t1.ibd
```

Bạn có thể sử dụng mệnh đề `DATA DIRECTORY `của câu lệnh `CREATE TABLE` để tự động tạo một tệp dữ liệu tablespace riêng cho mỗi bảng, nằm ngoài thư mục dữ liệu. Để biết thêm thông tin, hãy xem [Section 17.6.1.2, “Creating Tables Externally”](https://dev.mysql.com/doc/refman/8.4/en/innodb-create-table-external.html "17.6.1.2 Creating Tables Externally").
