Những lời khuyên về hiệu suất này bổ sung cho các hướng dẫn chung về chèn nhanh trong [Section 10.2.5.1, “Optimizing INSERT Statements”](https://dev.mysql.com/doc/refman/9.7/en/insert-optimization.html "10.2.5.1 Optimizing INSERT Statements").

Khi nhập dữ liệu vào InnoDB, hãy tắt chế độ tự động commit (autocommit), vì chế độ này sẽ ghi nhật ký vào đĩa cho mỗi lần chèn dữ liệu. Để vô hiệu hóa chế độ tự động commit trong quá trình nhập dữ liệu, hãy bao quanh nó bằng các câu lệnh SET autocommit và COMMIT:

```sql
SET autocommit=0;
... SQL import statements ...
COMMIT;
```

Tùy chọn mysqldump --opt tạo ra các tập tin sao lưu có tốc độ nhập nhanh vào bảng InnoDB, ngay cả khi không sử dụng các câu lệnh SET autocommit và COMMIT để bao bọc chúng.

Nếu bạn có ràng buộc UNIQUE trên các khóa phụ, bạn có thể tăng tốc quá trình nhập bảng bằng cách tạm thời tắt kiểm tra tính duy nhất trong suốt quá trình nhập:

```sql
SET unique_checks=0;
... SQL import statements ...
SET unique_checks=1;
```

Đối với các bảng lớn, điều này giúp tiết kiệm rất nhiều thao tác đọc/ghi ổ đĩa vì InnoDB có thể sử dụng bộ đệm thay đổi của nó để ghi các bản ghi secondary index theo batch. Hãy chắc chắn rằng dữ liệu không chứa các khóa trùng lặp.

Nếu bảng của bạn có ràng buộc khóa ngoại, bạn có thể tăng tốc quá trình nhập bảng bằng cách tắt kiểm tra khóa ngoại trong suốt phiên nhập dữ liệu:

```sql
SET foreign_key_checks=0;
... SQL import statements ...
SET foreign_key_checks=1;
```

Đối với các bảng dữ liệu lớn, điều này có thể tiết kiệm được rất nhiều thao tác đọc/ghi ổ đĩa.

Hãy sử dụng cú pháp INSERT nhiều hàng để giảm thiểu chi phí giao tiếp giữa máy khách và máy chủ nếu bạn cần chèn nhiều hàng:

```sql
INSERT INTO yourtable VALUES (1,2), (5,5), ...;
```

Mẹo này áp dụng cho việc chèn dữ liệu vào bất kỳ bảng nào, không chỉ riêng bảng InnoDB.

Khi thực hiện chèn dữ liệu hàng loạt vào các bảng có cột tự động tăng, hãy đặt innodb_autoinc_lock_mode thành 2 (xen kẽ) thay vì 1 (liên tiếp). Xem [Section 17.6.1.6, “AUTO_INCREMENT Handling in InnoDB”](https://dev.mysql.com/doc/refman/9.7/en/innodb-auto-increment-handling.html "17.6.1.6 AUTO_INCREMENT Handling in InnoDB") để biết thêm chi tiết.

Khi thực hiện chèn dữ liệu hàng loạt, việc chèn các hàng theo thứ tự KHÓA CHÍNH sẽ nhanh hơn. Các bảng InnoDB sử dụng chỉ mục [clustered index](https://dev.mysql.com/doc/refman/9.7/en/glossary.html#glos_clustered_index "clustered index"), giúp việc sử dụng dữ liệu theo thứ tự KHÓA CHÍNH diễn ra tương đối nhanh. Việc thực hiện chèn dữ liệu hàng loạt theo thứ tự KHÓA CHÍNH đặc biệt quan trọng đối với các bảng không nằm gọn hoàn toàn trong vùng bộ nhớ đệm.

Để đạt hiệu suất tối ưu khi tải dữ liệu vào chỉ mục FULLTEXT của InnoDB, hãy làm theo các bước sau:

a. Khi tạo bảng, hãy định nghĩa một cột FTS_DOC_ID, có kiểu dữ liệu BIGINT UNSIGNED NOT NULL, với một chỉ mục duy nhất có tên là FTS_DOC_ID_INDEX. Ví dụ:

```sql
CREATE TABLE t1 (
    FTS_DOC_ID BIGINT unsigned NOT NULL AUTO_INCREMENT,
    title varchar(255) NOT NULL DEFAULT '',
    text mediumtext NOT NULL,
PRIMARY KEY (`FTS_DOC_ID`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE UNIQUE INDEX FTS_DOC_ID_INDEX on t1(FTS_DOC_ID);
```

b. Tải dữ liệu vào bảng.

c. Tạo chỉ mục FULLTEXT sau khi dữ liệu được tải.

>[!note]
>Khi thêm cột FTS_DOC_ID tại thời điểm tạo bảng, hãy đảm bảo rằng cột FTS_DOC_ID được cập nhật khi cột được lập chỉ mục FULLTEXT được cập nhật, vì FTS_DOC_ID phải tăng dần đều với mỗi thao tác INSERT hoặc UPDATE. Nếu bạn chọn không thêm FTS_DOC_ID tại thời điểm tạo bảng và để InnoDB quản lý ID tài liệu cho bạn, InnoDB sẽ thêm FTS_DOC_ID dưới dạng cột ẩn với lệnh CREATE FULLTEXT INDEX tiếp theo. Tuy nhiên, cách tiếp cận này yêu cầu xây dựng lại bảng, điều này có thể ảnh hưởng đến hiệu suất.

Nếu tải dữ liệu vào một phiên bản MySQL mới, hãy cân nhắc tắt tính năng ghi nhật ký redo bằng cú pháp ALTER INSTANCE {ENABLE|DISABLE} INNODB REDO_LOG. Việc tắt tính năng ghi nhật ký redo giúp tăng tốc độ tải dữ liệu bằng cách tránh ghi vào nhật ký redo. Để biết thêm thông tin, hãy xem phần [Disabling Redo Logging](https://dev.mysql.com/doc/refman/9.7/en/innodb-redo-log.html#innodb-disable-redo-logging "Disabling Redo Logging").

>[!warning]
>Tính năng này chỉ dành cho việc tải dữ liệu vào một phiên bản MySQL mới. Không được tắt tính năng ghi nhật ký phục hồi (redo logging) trên hệ thống sản xuất. Có thể tắt và khởi động lại máy chủ khi tính năng ghi nhật ký phục hồi bị tắt, nhưng việc máy chủ dừng đột ngột khi tính năng này bị tắt có thể gây mất dữ liệu và làm hỏng phiên bản MySQL.

Sử dụng MySQL Shell để nhập dữ liệu. Tiện ích nhập bảng song song `util.importTable() `của MySQL Shell cung cấp khả năng nhập dữ liệu nhanh chóng vào bảng quan hệ MySQL đối với các tệp dữ liệu lớn. Tiện ích tải dữ liệu `util.loadDump() `của MySQL Shell cũng cung cấp khả năng tải song song. Xem [MySQL Shell Utilities](https://dev.mysql.com/doc/mysql-shell/9.7/en/mysql-shell-utilities.html).
