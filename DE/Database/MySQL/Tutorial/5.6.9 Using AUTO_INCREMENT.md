Thuộc tính AUTO_INCREMENT có thể được sử dụng để tạo định danh duy nhất cho các hàng mới.

```sql
CREATE TABLE animals ( 
	id MEDIUMINT NOT NULL AUTO_INCREMENT, 
	name CHAR(30) NOT NULL, 
	PRIMARY KEY (id) 
); 
INSERT INTO animals (name) VALUES 
	('dog'),('cat'),('penguin'), 
	('lax'),('whale'),('ostrich'); 

SELECT * FROM animals;
```

Điều này trả về

```none
+----+---------+
| id | name    |
+----+---------+
|  1 | dog     |
|  2 | cat     |
|  3 | penguin |
|  4 | lax     |
|  5 | whale   |
|  6 | ostrich |
+----+---------+
```

Không có giá trị nào được chỉ định cho cột AUTO_INCREMENT, vì vậy MySQL tự động gán số thứ tự. Bạn cũng có thể gán rõ ràng giá trị 0 cho cột để tạo số thứ tự, trừ khi chế độ SQL NO_AUTO_VALUE_ON_ZERO được bật. Ví dụ:

```sql
INSERT INTO animals (id,name) VALUES(0,'groundhog');
```

Nếu cột được khai báo là NOT NULL, bạn cũng có thể gán giá trị NULL cho cột đó để tạo số thứ tự. Ví dụ:

```sql
INSERT INTO animals (id,name) VALUES(NULL,'squirrel');
```

Khi bạn chèn bất kỳ giá trị nào khác vào cột AUTO_INCREMENT, cột đó sẽ được đặt thành giá trị đó và trình tự sẽ được đặt lại sao cho giá trị được tạo tự động tiếp theo sẽ theo thứ tự từ giá trị lớn nhất của cột. Ví dụ:

```sql
INSERT INTO animals (id,name) VALUES(100,'rabbit');
INSERT INTO animals (id,name) VALUES(NULL,'mouse');
SELECT * FROM animals;
+-----+-----------+
| id  | name      |
+-----+-----------+
|   1 | dog       |
|   2 | cat       |
|   3 | penguin   |
|   4 | lax       |
|   5 | whale     |
|   6 | ostrich   |
|   7 | groundhog |
|   8 | squirrel  |
| 100 | rabbit    |
| 101 | mouse     |
+-----+-----------+
```

Việc cập nhật giá trị của cột AUTO_INCREMENT hiện có cũng sẽ đặt lại trình tự AUTO_INCREMENT.

Bạn có thể truy xuất giá trị AUTO_INCREMENT được tạo tự động gần đây nhất bằng hàm `SQL LAST_INSERT_ID()` hoặc hàm API C `mysql_insert_id()`. Các hàm này chỉ dành riêng cho một kết nối cụ thể, do đó giá trị trả về của chúng không bị ảnh hưởng bởi một kết nối khác cũng đang thực hiện thao tác chèn.

Hãy sử dụng kiểu dữ liệu số nguyên nhỏ nhất cho cột AUTO_INCREMENT đủ lớn để chứa giá trị số thứ tự tối đa mà bạn yêu cầu. Khi cột đạt đến giới hạn trên của kiểu dữ liệu, lần tạo số thứ tự tiếp theo sẽ thất bại. Nếu có thể, hãy sử dụng thuộc tính UNSIGNED để cho phép phạm vi lớn hơn. Ví dụ, nếu bạn sử dụng TINYINT, số thứ tự tối đa cho phép là 127. Đối với TINYINT UNSIGNED, mức tối đa là 255. Xem [Section 13.1.2, “Integer Types (Exact Value) - INTEGER, INT, SMALLINT, TINYINT, MEDIUMINT, BIGINT”](https://dev.mysql.com/doc/refman/8.4/en/integer-types.html "13.1.2 Integer Types (Exact Value) - INTEGER, INT, SMALLINT, TINYINT, MEDIUMINT, BIGINT") để biết phạm vi của tất cả các kiểu số nguyên.

>[!note]
>Đối với thao tác chèn nhiều hàng, các hàm LAST_INSERT_ID() và mysql_insert_id() thực tế trả về khóa AUTO_INCREMENT từ hàng đầu tiên được chèn. Điều này cho phép các thao tác chèn nhiều hàng được sao chép chính xác trên các máy chủ khác trong thiết lập replication.

Để bắt đầu với giá trị AUTO_INCREMENT khác 1, hãy đặt giá trị đó bằng lệnh CREATE TABLE hoặc ALTER TABLE, như sau:

```sql
mysql> ALTER TABLE tbl AUTO_INCREMENT = 100;
```

#### MyISAM Notes

Đối với các bảng MyISAM, bạn có thể chỉ định AUTO_INCREMENT trên một cột phụ trong chỉ mục nhiều cột. Trong trường hợp này, giá trị được tạo cho cột AUTO_INCREMENT được tính bằng `MAX(auto_increment_column) + 1 WHERE prefix=given-prefix`. Điều này hữu ích khi bạn muốn nhóm dữ liệu theo thứ tự.

```sql
CREATE TABLE animals (
    grp ENUM('fish','mammal','bird') NOT NULL,
    id MEDIUMINT NOT NULL AUTO_INCREMENT,
    name CHAR(30) NOT NULL,
    PRIMARY KEY (grp,id)
) ENGINE=MyISAM;

INSERT INTO animals (grp,name) VALUES
    ('mammal','dog'),('mammal','cat'),
    ('bird','penguin'),('fish','lax'),('mammal','whale'),
    ('bird','ostrich');

SELECT * FROM animals ORDER BY grp,id;
```

Kết quả trả về là:

```none
+--------+----+---------+
| grp    | id | name    |
+--------+----+---------+
| fish   |  1 | lax     |
| mammal |  1 | dog     |
| mammal |  2 | cat     |
| mammal |  3 | whale   |
| bird   |  1 | penguin |
| bird   |  2 | ostrich |
+--------+----+---------+
```

Trong trường hợp này (khi cột AUTO_INCREMENT là một phần của chỉ mục nhiều cột), các giá trị AUTO_INCREMENT sẽ được sử dụng lại nếu bạn xóa hàng có giá trị AUTO_INCREMENT lớn nhất trong bất kỳ nhóm nào. Điều này xảy ra ngay cả với các bảng MyISAM, mà thông thường các giá trị AUTO_INCREMENT không được sử dụng lại.

Nếu cột AUTO_INCREMENT là một phần của nhiều chỉ mục, MySQL sẽ tạo các giá trị chuỗi bằng cách sử dụng chỉ mục bắt đầu bằng cột AUTO_INCREMENT, nếu có. Ví dụ, nếu bảng animals chứa các chỉ mục PRIMARY KEY (grp, id) và INDEX (id), MySQL sẽ bỏ qua PRIMARY KEY khi tạo các giá trị chuỗi. Kết quả là, bảng sẽ chỉ chứa một chuỗi duy nhất, chứ không phải một chuỗi cho mỗi giá trị grp.
